\section{Obiettivo}
Trasformare la Todo App (client-only) in un'applicazione Full-Stack. Il backend, scritto in Node.js con Express o Python con Flask, gestirà la persistenza dei dati su file JSON e garantirà l'accesso protetto tramite Basic Authentication.

\section{Setup del Progetto}
\begin{enumerate}
    \item Creare una cartella \texttt{src/lezione4/server}.
    \item \textbf{Node.js}: Inizializzare con \texttt{npm init -y} e installare \texttt{express express-basic-auth}.
    \item \textbf{Python}: Creare un ambiente virtuale e installare:
    \begin{lstlisting}[language=bash]
pip install flask flask-basicauth
    \end{lstlisting}
\end{enumerate}

\section{Task 1: Backend}
Implementare le rotte CRUD su \texttt{app.js} (Node) o \texttt{app.py} (Python) e proteggerle con il middleware di autenticazione.

\section{Task 2: Frontend}
Modificare la \texttt{index.html} (es. versione con Tailwind) per utilizzare \texttt{fetch} con header \texttt{Authorization}.

\section{Task Bonus: Funzionalità Avanzate}
\begin{enumerate}
    \item \textbf{Gestione Priorità}: Aggiungere un menù a tendina nel form (es. Bassa, Media, Alta) e aggiornare il modello dei dati per includere il campo \texttt{priorità} nel task.
    \item \textbf{Pulsante Backup}: Aggiungere un pulsante nell'interfaccia che, tramite una rotta dedicata (es. \texttt{POST /backup}), forza il salvataggio immediato dello stato corrente su un nuovo file (es. \texttt{backup.json}).
\end{enumerate}
